# RHIR 常見欄位草案

更新日期：2026-05-10

## 一、用途

本文件用來整理 RHIR，Rental Housing Interoperability Resources，第一版可能需要的常見欄位。

目前重點不是決定最終 JSON 格式，而是先整理租屋資訊中常見、重要、可被結構化的欄位。後續可再根據實作需求，進一步討論欄位命名、資料型別、必要欄位、選填欄位與揭露狀態。

## 二、整理原則

RHIR 不只是記錄「房子有什麼」，而是記錄：

- 誰提供了這項資訊。
- 這項資訊從哪裡來。
- 這項資訊揭露到什麼程度。
- 是否存在不同來源之間的資訊衝突。
- 還有哪些資訊目前不可知。

因此，欄位設計應同時考慮房屋本體、位置、費用、設備、租約條件、管理、提供者身份、法定狀態與資訊透明度。

## 三、建議第一層架構

RHIR 第一版可先分成以下區塊：

- metadata
- listing
- property
- layoutAndStructure
- locationContext
- amenities
- leaseTerms
- cost
- management
- provider
- ownershipLegalStatus
- marketingClaims
- transparencyLayer

## 四、Metadata 資料紀錄基本識別

此區用來記錄 RHIR 本身的基本資訊，讓每一筆資料可以被追蹤、更新與比較。

### 常見欄位

- RHIR record ID
- RHIR version
- 建立日期
- 更新日期
- 資料擷取日期
- 資料來源平台
- 來源網址
- 原始平台物件 ID
- 資料狀態：刊登中 / 已下架 / 未知

## 五、Listing 平台刊登資訊

此區記錄租屋平台或刊登頁面直接呈現的物件資訊。

### 常見欄位

- 物件標題
- 物件副標
- 平台標籤
- 刊登類型
- 是否急租
- 是否精選或廣告物件
- 是否由屋主刊登
- 是否由仲介刊登

### 範例

- 秒殺精裝房
- 寵物友善
- 垃圾代收
- 拎包入住
- 近捷運
- 近商圈
- 隨時可遷入

## 六、Property 房屋基本資料

此區記錄房屋本體與物件基本條件。

### 常見欄位

- 城市
- 行政區
- 地址
- 所屬社區
- 房屋類型
- 租屋類型
- 法定用途
- 屋齡
- 坪數
- 樓層
- 總樓高
- 是否頂樓
- 產權登記狀態
- 建物面積
- 使用執照資訊

### 範例

- 社區：友座森觀
- 類型：整層住家 / 電梯大廈
- 法定用途：住家用
- 屋齡：16 年
- 坪數：55 坪
- 樓層 / 樓高：11 / 12 樓
- 產權登記：有

## 七、Layout and Structure 格局與建物結構

此區記錄空間配置、朝向與隔間條件。

### 常見欄位

- 格局
- 房間數
- 客廳數
- 衛浴數
- 陽台數
- 廚房數
- 朝向
- 隔間材質
- 是否有陽台
- 是否有獨立門牌
- 是否乾濕分離
- 是否有對外窗
- 通風資訊
- 採光資訊

### 範例

- 格局：3 房 2 廳 2 衛 - 1 陽台 1 廚房
- 朝向：坐西朝東
- 隔間材質：水泥

## 八、Location Context 位置與周邊環境

此區記錄位置、交通、生活機能與教育資源。這些資料可來自平台揭露，也可由公開資料或系統主動搜尋補足。

### 交通欄位

- 最近捷運站
- 最近捷運站距離
- 最近公車站
- 最近公車站距離
- 公車路線密度
- Ubike 站點距離
- 到學校或工作地點通勤時間
- 夜間交通便利性

### 生活機能欄位

- 便利商店距離
- 超市距離
- 傳統市場距離
- 百貨公司距離
- 餐廳數量
- 購物中心數量
- 公園綠地距離
- 醫療機構距離
- 夜市距離
- 洗衣店距離

### 教育欄位

- 最近國小
- 最近國中
- 最近高中職
- 最近大學
- 附近國小數量
- 附近國中數量
- 附近大學數量

## 十、Lease Terms 租住條件與守則

此區記錄租期、入住時間、使用限制與租客條件。

### 常見欄位

- 最短租期
- 是否允許短期租賃
- 可遷入日
- 是否可隨時遷入
- 是否可開伙
- 是否可養寵物
- 是否禁菸
- 性別要求
- 身分要求
- 是否需工作證明
- 房東是否同住
- 是否可報稅
- 是否可遷戶籍
- 提前解約條款
- 修繕責任
- 押金退還條件

### 範例

- 最短租期：一年
- 短期租賃：不可以
- 開伙：可
- 可遷入日：2026-05-01
- 飼養寵物：不可
- 性別要求：男女不拘
- 身分要求：不限身份
- 房東同住：否
- 室內禁止抽菸：是
- 需有工作證明：是

## 十一、Cost 費用與金流條件

此區專門記錄與金錢有關的資訊，避免費用散落在描述文字中。

### 常見欄位

- 月租金
- 押金
- 押金月數
- 管理費
- 水費
- 電費
- 瓦斯費
- 網路費
- 第四台費用
- 清潔費
- 垃圾代收費
- 寵物額外押金
- 寵物清潔費
- 仲介服務費
- 車位費
- 其他費用

### 資訊衝突範例

同一物件可能出現：

- 平台欄位：押金面議
- 仲介內文：押金 2 個月

此時 RHIR 不應直接選擇其中一個，而應記錄為資訊衝突。

## 十二、Management 管理與社區服務

此區記錄社區、管理方式與公共服務。

### 常見欄位

- 社區名稱
- 管理方式
- 是否有管理員
- 是否有警衛
- 是否 24 小時管理
- 管理費
- 管理費描述
- 垃圾處理方式
- 公共空間
- 停車管理

### 範例

- 社區：友座森觀
- 管理方式：管理員 / 警衛
- 管理費：6075 元 / 月
- 垃圾處理：B1 垃圾回收場

## 十三、Provider 物件提供者與可追溯身份

此區記錄資訊提供者身份，類似 FHIR 中用來辨識角色或組織的概念。租屋風險不只來自物件，也來自資訊提供者是否可追溯。

### 常見欄位

- 提供者類型：屋主 / 仲介 / 代管 / 平台
- 仲介姓名
- 房東姓名或代稱
- 是否收服務費
- 經紀業名稱
- 加盟店名稱
- 經紀人證號
- 營業員證號
- 聯絡方式
- LINE 聯絡方式
- 電話
- 是否可驗證

### 範例

- 仲介：黃先生
- 是否收服務費：是
- 經紀業：乙尚村房地產仲介業有限公司
- 加盟店：台灣房屋板橋府中特許加盟店
- 經紀人證號：95 北縣登字第001288號
- 營業員證號：110 登字第389673號

## 十四、Ownership and Legal Status 產權與法定狀態

此區記錄產權、用途與法定狀態。此類資料目前先以平台揭露為主，未來可再接入政府或不動產正式資料。

### 常見欄位

- 產權登記
- 房屋是否已辦產權登記
- 法定用途
- 使用執照資訊
- 建物面積
- 是否可查驗
- 是否疑似違建
- 是否疑似頂樓加蓋
- 是否疑似工業住宅

### 範例

- 產權登記：有
- 法定用途：住家用
- 使用執照資訊：未揭露或見使照

## 十五、Marketing Claims 廣告描述與主張

此區記錄平台或刊登者的描述性文字。這些內容應被保存，但不應直接視為客觀事實。

### 常見欄位

- 廣告標題
- 特色描述
- 優選理由
- 物件亮點
- 裝潢描述
- 交通主張
- 安全主張
- 學區主張
- 生活機能主張
- 可立即入住主張

### 範例

- 秒殺精裝房
- 出行超便利
- 產權有保障
- 窗外綠意山景
- 明星學區
- 一卡皮箱入住
- 安全無虞

### 注意事項

- 「近捷運」應拆成實際距離與來源。
- 「出行超便利」應視為行銷主張。
- 「安全無虞」應視為行銷主張，不可直接等同於客觀安全。

## 十六、Transparency Layer 資訊透明度層

此區記錄欄位是否揭露、是否完整、是否有衝突，以及是否可由公開資料補足。

### 常見欄位

- 已揭露欄位數
- 部分揭露欄位數
- 未揭露欄位數
- 資訊衝突欄位數
- 可由公開資料補足欄位數
- 可由系統推估欄位數
- 不可判斷欄位數
- 需使用者補問欄位

### 常見未揭露項目

- 是否可報稅
- 是否可遷戶籍
- 提前解約條款
- 修繕責任
- 押金退還條件
- 寵物是否需額外押金
- 垃圾代收是否另收費
- 實際使用執照內容
- 對外窗資訊
- 通風資訊

## 十八、後續討論事項

後續需要再討論：

- 哪些欄位是 RHIR 必填。
- 哪些欄位是選填。
- 欄位命名要使用中文、英文或雙語。
- 欄位值要如何標準化。
- 揭露狀態如何和欄位綁定。
- 資訊來源如何記錄。
- 資訊衝突如何呈現。
- 何時進入正式 JSON schema 設計。
