# Rent Unfiltered 透明租屋：專案整理

更新日期：2026-05-09

## 一、專案名稱

- 專案名稱：Rent Unfiltered
- 中文名稱：透明租屋

### 副標候選

- 看懂租約、費用與風險的 AI 租屋檢查工具
- 讓租屋不再靠運氣，讓風險變得看得見
- 不懂租約，也能看懂風險

## 二、專案定位

Rent Unfiltered 不是另一個租屋刊登平台，也不是主觀評論平台，而是一個面向租屋初心者的租屋資訊整合、風險判讀與報告生成系統。

本專案希望處理的不是「如何讓使用者看到更多物件」，而是「如何讓使用者真正看懂一個物件」。對學生、新鮮人與第一次租屋者而言，租屋風險往往不只來自價格，而是來自契約條款、費用揭露、居住安全、租客權益與資訊不完整。

因此，Rent Unfiltered 的核心目標是將原本分散、模糊、難以比較的租屋資訊，轉化為可閱讀、可比較、可追溯、可解釋的決策資訊，降低租屋資訊不對稱與搜尋成本。

### 定位定義

- 產品類型：租屋資訊透明化與風險判讀系統
- 主要對象：學生、新鮮人、第一次租屋者與資訊弱勢租客
- 核心任務：把分散的租屋資訊整理成可理解、可比較、可追溯的風險資訊
- 最終價值：讓租屋決策不只看價格與曝光，而能看見價格背後的風險

### 非目標

- 不提供租屋刊登與流量分發
- 不以主觀評論作為核心資料來源
- 不替使用者做最終決定
- 不提供法律保證、交易背書或安全保證

## 三、最終版本產品樣貌

最終版本的 Rent Unfiltered 會是一個以單一物件為中心的租屋風險分析系統。

使用者可以輸入或匯入一個租屋物件的資料來源，例如平台頁面、租約文字、房東提供資訊、公開資料或其他補充內容。系統會先將不同來源的資料整理成統一格式，再標示每個欄位的來源、揭露程度、衝突狀態與可信度，最後生成一份可供閱讀與比較的租屋風險報告。

這個產品的最終體驗，不是只給一個分數，而是讓使用者同時看到：

- 這個物件目前有哪些資訊已經清楚揭露
- 哪些資訊仍然未知或不完整
- 哪些資訊之間彼此矛盾
- 哪些條件可能構成契約、費用、安全或權益風險
- 哪些問題需要進一步詢問房東、仲介或自行查核

## 四、核心功能

### 1. 租屋資訊整合

- 接收來自不同來源的租屋資訊
- 整理平台欄位、租約條款、補充說明與公開資料
- 將分散資訊合併成同一份物件資料

### 2. RHIR 結構化標準化

- 將租屋資訊轉換為 RHIR，Rental Housing Interoperability Resources
- 以一致格式記錄房屋、契約、費用、設備、權益與環境資訊
- 保留每個欄位的來源、原文依據、揭露狀態與更新資訊

### 3. Transparency Layer 資訊透明層

- 記錄欄位是否已揭露、部分揭露、未揭露或無法判斷
- 記錄資訊來源是平台、房東、仲介、公開資料、系統推估或來源未知
- 記錄不同來源之間是否存在資訊衝突
- 區分「未揭露」與「不存在」，避免錯誤推論

### 4. 風險分析引擎

- 根據結構化資料計算 Rental Risk Index，RRI
- 依面向分析契約、費用、安全與權益風險
- 將風險分數與風險原因一起輸出，而不是只回傳單一結果

### 5. AI 輔助判讀

- 白話翻譯租約條款與複雜描述
- 標示可能需要特別注意的風險內容
- 協助產生追問房東或仲介的問題
- 生成使用者可直接閱讀的摘要與建議

### 6. 租屋風險報告

- 輸出單一物件的完整分析結果
- 提供可閱讀、可比較、可追溯的報告內容
- 支援後續保存、比較、匯出與討論

## 五、系統架構

Rent Unfiltered 的最終版本可分為五層：

### 1. Input Layer

- 使用者輸入資料
- 租約文字或檔案
- 平台物件資訊
- 外部公開資料

### 2. RHIR Normalization Layer

- 將不同來源資料轉成統一欄位格式
- 建立單一物件的 RHIR 記錄
- 保留欄位原始內容與標準化結果

### 3. Transparency Layer

- 記錄來源
- 記錄揭露狀態
- 記錄衝突
- 記錄可推估與不可判斷資訊

### 4. Risk Engine Layer

- 根據 RHIR 欄位進行風險分析
- 產生 RRI 分數、風險等級與分面向結果
- 將資訊不確定性與實際風險分開處理

### 5. Report and Explanation Layer

- 生成白話摘要
- 生成風險原因說明
- 生成簽約前檢查清單
- 生成追問建議與後續查核方向

## 六、資料架構

RHIR 是 Rent Unfiltered 的核心資料格式。它不只是記錄「房子有什麼」，而是記錄一筆租屋資訊如何被提供、如何被揭露、如何被判讀。

一份 RHIR 至少應包含以下結構：

- `property`：房屋基本資料
- `location`：位置與區域資訊
- `pricing`：租金與各項費用
- `contract`：契約條件
- `facilities`：設備與家具
- `safety`：安全相關資訊
- `rights`：租客權益與限制
- `provider`：資訊提供者身份
- `sources`：資料來源清單
- `transparency`：揭露、衝突、未知與推估狀態
- `riskScore`：風險分數與面向分數
- `aiFindings`：AI 分析結果
- `recommendations`：建議事項

每個重要欄位都應保留：

- 欄位值是什麼
- 這個值來自哪個來源
- 這個值的揭露程度如何
- 是否存在衝突或不確定性
- 是否需要後續人工確認

## 七、風險分析架構

RRI，Rental Risk Index，是 Rent Unfiltered 的核心分析模型，但它是整體系統中的一個模組，不是整個產品本身。

RRI 的目的不是替使用者做決定，而是提供一個可解釋的風險參考維度，讓租屋判斷不只停留在價格、地點或平台包裝，而能納入實際風險成本。

### 主要風險面向

- 契約透明度
- 費用透明度
- 居住安全
- 租客權益
- 生活適配度

### 契約透明度

- 是否有書面契約
- 是否有審閱期
- 是否明確約定修繕責任
- 是否明確約定提前解約條件
- 是否明確約定押金退還方式

### 費用透明度

- 租金
- 押金
- 水電費
- 管理費
- 網路費
- 清潔費
- 其他額外費用

### 居住安全

- 是否頂樓加蓋
- 是否違法隔間
- 逃生動線
- 消防設備
- 漏水狀況
- 電線安全
- 門鎖與出入安全

### 租客權益

- 是否禁止報稅
- 是否禁止遷戶籍
- 是否有稅負轉嫁
- 是否存在違法或不合理條款

### 生活適配度

- 通勤便利性
- 噪音
- 採光
- 垃圾處理
- 室友或鄰居條件
- 寵物限制

### 分數表示

- 0-20：低風險
- 21-40：中低風險
- 41-60：中風險
- 61-80：高風險
- 81-100：極高風險

### 分析原則

- 透明度分數與風險分數應分開處理
- 未揭露不等於不存在
- 生活適配度應避免與客觀風險完全混合
- 每個風險結果都應能回溯到資料來源與判斷依據

## 八、AI Agent 架構

AI 在 Rent Unfiltered 中不是最終決定者，而是資訊整理、風險揭露與白話轉譯的輔助系統。

### 1. 租約分析 Agent

- 分析租約中的風險條款
- 找出模糊、不完整或不利租客的描述
- 將條款翻譯成一般人可理解的白話內容

### 2. 費用透明度 Agent

- 分析租金與各種附加費用
- 標示費用是否清楚揭露
- 判斷是否可能存在不透明費用

### 3. 地區行情 Agent

- 比對區域租金行情
- 提供每坪租金或同類型物件的相對位置
- 協助辨識異常偏高或偏低的價格

### 4. 報告生成 Agent

- 整理風險重點
- 生成白話摘要
- 生成簽約前檢查項目
- 生成後續應追問與查核的建議

## 九、報告輸出

Rent Unfiltered 的最終輸出是一份單一物件報告，而不是只有分數頁面。

報告內容應包含：

- 總風險分數
- 風險等級
- 分面向分數
- 主要風險原因
- 白話解釋
- 資訊揭露狀態摘要
- 衝突欄位摘要
- 簽約前建議
- 待追問事項
- 待自行查核事項
- 條款逐條分析
- 租金行情比較

## 十、資料來源規劃

Rent Unfiltered 的資料來源可分為三類：

### 1. 主動揭露資料

- 房東提供資訊
- 仲介提供資訊
- 平台刊登欄位
- 使用者自行輸入資料

### 2. 公開資料

- 政府開放資料
- 內政部實價登錄租賃資料
- 內政部住宅租賃糾紛統計
- g0v 開放台灣民間租屋資料

### 3. 系統整合或推導資料

- 租約文字分析結果
- 租金行情比對結果
- 系統根據結構化資料產生的推估欄位

所有外部資料都應保留來源、授權、更新頻率與可追溯資訊，不應在未標示來源的情況下直接混入最終判斷。

## 十一、系統邊界與設計原則

### 系統邊界

- 不提供租屋刊登功能
- 不替使用者做最終租屋決定
- 不提供法律判決或安全保證
- 不以主觀評論作為核心資料來源

### 設計原則

- 透明的規則
- 可解釋的判斷
- 可被檢視的資料
- 可持續修正的風險模型
- 可保存、可交換、可比較的資訊格式

## 十二、最終目標

Rent Unfiltered 希望讓租屋不再只靠運氣、話術與曝光，而是讓資訊品質本身參與競爭。

它要解決的不是單純的搜尋問題，而是租屋資訊如何被量化、交換、解釋與信任的問題。當安全、契約透明、費用透明與租客權益能被更清楚地揭露與比較時，租屋市場就有機會從依賴經驗與包裝，走向依賴資訊品質與風險理解。

## 十三、可納入最終版的延伸參考

本節為獨立保留的延伸方向，不構成目前主架構的一部分，但未來可在不改變專案定位的前提下併入最終版本。

### 1. 更聚焦的早期目標使用場景

- 以台北校外租屋的大學生作為第一批驗證對象
- 以政大學生在文山區、木柵一帶找房作為早期測試場景
- 作為使用者訪談、資料蒐集與報告設計的優先場域

### 2. 更具體的資訊分散情境描述

- 補充目前常見來源，如 591、學校校外租屋網、Facebook 租屋社團、LINE 群組與仲介私訊
- 作為「資訊分散」問題的具體例證
- 用於簡報、企劃書與訪談問題設計

### 3. 可直接納入 RHIR 的重要欄位

- 是否可租補
- 是否能報稅
- 是否可養寵物
- 樓層與屋況
- 其他會明顯影響租屋判斷的條件欄位

### 4. 地區試點與驗證策略

- 先以單一學區或單一行政區作為試點
- 集中建立初步樣本、測試風險報告與驗證使用流程
- 作為最終版本擴展前的場域驗證方法

### 5. 未來商業化延伸方向

- 作為仲介、包租代管或租屋平台的風險揭露與報告生成工具
- 提供租約分析、資訊標準化與報告輸出能力
- 以分析層與標準層角色切入，而非直接轉型為租屋刊登平台
