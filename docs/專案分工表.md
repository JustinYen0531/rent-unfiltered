# Rent Unfiltered 透明租屋：專案分工表

更新日期：2026-05-09

## 一、分工原則

本專案採取「產品技術落地」與「市場風險研究」雙主軸分工。

一方負責把想法做成可展示、可操作、可輸出的技術 Demo；另一方負責讓專案的問題定義、風險模型、資料依據與市場意義更加完整。

這份分工不是固定死的工作清單，而是目前討論階段的參考分配。後續可以依照比賽需求、開發進度與組員擅長領域調整。

## 二、我：產品 / 技術主導

### 角色定位

負責讓 Rent Unfiltered 從概念變成可以實際操作的技術 Demo。

主要任務是把租屋風險檢查、AI 分析、報告生成、資料格式與使用者介面落地，讓作品可以被展示、被操作、被理解。

### 主要負責內容

- AI Agent
- 前端介面
- JSON schema
- 系統架構
- 租約分析功能
- 報告生成功能
- Demo 流程
- UI 設計
- Pitch 視覺化

### 具體工作方向

- 建立使用者輸入租屋條件的表單。
- 設計租屋風險分數的前端呈現方式。
- 將風險結果轉成白話提醒。
- 製作租屋檢查清單與建議區塊。
- 規劃 AI Agent 如何分析租約文字。
- 設計租屋資訊 JSON 或 RentRisk JSON 的資料結構。
- 建立可展示的 Demo 流程。
- 將組員整理出的風險規則與資料依據整合進系統。
- 製作簡報中需要的產品畫面、流程圖與展示素材。

### 核心責任

- 讓專案做得出來。
- 讓 Demo 能順利展示。
- 讓使用者可以看懂操作流程。
- 讓 AI 分析結果可以被視覺化與報告化。

## 三、組員：市場 / 風險 / 分析主導

### 角色定位

負責讓 Rent Unfiltered 的問題定義、租屋風險模型與市場分析站得住腳。

主要任務是從經濟、市場、租屋行為、資訊不對稱與使用者研究角度，協助判斷這個工具應該分析什麼、為什麼這樣分析、哪些資料可以支持判斷。

### 主要負責內容

- Rental Risk Index 設計
- 租屋風險權重討論
- 租金合理性分析
- 青年租屋負擔分析
- 資訊不對稱問題整理
- 使用者研究
- 租屋踩雷案例整理
- 政府資料或公開資料查找
- 簡報中的問題定義
- 專案的市場意義與社會價值說明

### 具體工作方向

- 討論哪些租屋因素應該納入風險分數。
- 協助判斷不同風險項目的嚴重程度。
- 查詢租屋糾紛、租屋法規、青年租屋負擔等相關資料。
- 整理 Dcard、PTT 或其他公開討論中的常見租屋痛點。
- 分析哪些問題最常造成租客損失或不安。
- 協助確認政府開放資料是否適合本專案使用。
- 討論租金合理性可以如何衡量。
- 協助建立風險指標的解釋邏輯。
- 負責簡報中「為什麼這個問題重要」的論述。

### 核心責任

- 讓專案說得通。
- 讓風險模型有依據。
- 讓問題定義明確。
- 讓作品不只是技術 Demo，而是有真實市場與使用者價值。

## 四、共同討論項目

以下內容需要兩人一起討論，不能完全由單方決定。

- Rental Risk Index 的最終面向。
- 每個風險面向的權重。
- 哪些條款或租屋條件算高風險。
- 風險分數如何分級。
- 是否納入生活適配度。
- 是否使用政府開放資料。
- 是否加入地區租金行情比較。
- 是否設計 RentRisk JSON 作為專案亮點。
- MVP 最終功能範圍。
- 比賽 Demo 的展示流程。
- 簡報主軸與故事線。

## 五、合作方式

### 我提供

- 可操作的產品原型。
- AI 分析與報告生成能力。
- 技術實作與 Demo 整合。
- 視覺化呈現與簡報素材。

### 組員提供

- 市場與資料分析。
- 風險模型依據。
- 使用者痛點整理。
- 問題定義與論述。
- 租屋議題的研究支持。

### 最終共同產出

- 一個能操作的 Rent Unfiltered 技術 Demo。
- 一套可解釋的租屋風險指數。
- 一份白話化租屋風險報告。
- 一個完整的比賽簡報故事。
- 一個能說明社會價值與市場需求的專案。

## 六、一句話總結

我負責讓 Rent Unfiltered 做得出來，組員負責讓 Rent Unfiltered 說得通、站得住，最後一起把它變成一個真正有使用者價值的作品。
