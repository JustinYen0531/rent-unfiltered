# Rent Unfiltered 三分鐘影片正式拍攝稿

本版採用「同一間房，資訊補齊前後的風險差異」作為主線。

判斷：目前不建議再加入新的大型展示段落。三分鐘影片最重要的是讓評審快速看懂 Rent Unfiltered 的核心價值，而不是展示所有頁面。真正值得保留的展示只有兩組：

1. 同一筆物件在資訊不足與補齊後的差異。
2. RHIR / RRI / AI 報告與資料保存，證明系統不只是概念。

Chrome 擴充插件、名詞解釋頁、資料庫管理頁可以作為快速帶過或備用畫面，不建議變成主段落。

## 影片總長

建議控制在 2 分 45 秒左右。

## 拍攝主線

591 物件頁面或截圖 -> Rent Unfiltered 首頁 -> 新增租屋資訊 -> 空白資訊版本 -> 建立第一版 -> 查看 RRI / RHIR -> 新增子版本 -> 一鍵補齊空白欄位 -> 重新查看 RRI / AI 報告 -> 上傳或查看 Supabase -> 結尾

## 0:00-0:20 開場

畫面：

- 顯示租屋網站頁面、租屋物件截圖或租屋資訊畫面。
- 快速切到 Rent Unfiltered 首頁。

旁白：

> 對很多學生和第一次租屋的人來說，租屋最大的問題不是找不到房子，而是看不懂風險。照片看起來可以，價格也在預算內，但水電費怎麼算、押金怎麼退、能不能報稅、修繕誰負責，這些真正重要的問題，常常不會直接寫清楚。

字幕標籤：

- 租屋資訊常常不完整
- 風險藏在條款與細節裡

## 0:20-0:45 匯入或建立一筆租屋資料

畫面：

- 點擊「新增租屋資訊」。
- 顯示表單一開始是空白。
- 可以手動填入少量基本欄位，或載入匯入 ID。

旁白：

> Rent Unfiltered 不是另一個租屋平台，而是一個租屋風險判讀工具。我們先把一筆租屋資訊放進系統，讓原本分散在頁面上的條件，變成可以被整理、比較和追蹤的資料。

字幕標籤：

- 新增租屋資訊
- 將租屋條件轉成可分析資料

## 0:45-1:15 第一次評估：資訊不足版本

畫面：

- 顯示表單仍有未填欄位。
- 建立版本 X。
- 進入詳細頁，顯示欄位完整度、待詢問欄位、RHIR 或 RRI。

旁白：

> 第一次評估時，我們刻意保留一些空白欄位，模擬真實租屋市場裡常見的狀況：物件看得到，但重要條款不一定完整。這時系統不會把空白當成安全，而是會把缺漏資訊也視為一種風險，提醒使用者哪些地方需要追問。

字幕標籤：

- 資訊不足也會形成風險
- 待追問欄位
- 初始版本 X

## 1:15-1:45 建立子版本並補齊資訊

畫面：

- 點「新增子版本」。
- 回到表單。
- 點左側「一鍵補齊空白欄位」。
- 顯示進度變成 100%、31/31。

旁白：

> 接著，我們建立同一筆物件的子版本，並補齊原本缺漏的欄位。這代表使用者在看房、詢問房東或比對契約後，可以把新資訊補回同一筆紀錄，而不是重新開始。系統也能保留前後版本，讓資訊變化可以被追蹤。

字幕標籤：

- 建立子版本
- 一鍵補齊空白欄位
- 31/31 欄位完整

## 1:45-2:20 第二次評估：RHIR / RRI / AI 報告

畫面：

- 建立補齊後的新版本。
- 顯示新版本的 RRI。
- 顯示 RHIR JSON 或欄位結構。
- 顯示 AI 白話風險報告。

旁白：

> 補齊資訊後，系統會重新生成 RHIR 結構與 RRI 風險判讀。使用者可以比較前後版本的差異，知道哪些風險只是因為資料不足造成，哪些風險即使補齊後仍然需要注意。AI 報告則會把技術性的欄位轉成比較好理解的租屋提醒。

字幕標籤：

- 重新生成 RHIR / RRI
- 比較前後版本
- AI 白話風險報告

## 2:20-2:40 資料保存與可追溯性

畫面：

- 上傳 RHIR 到 Supabase，或顯示資料庫頁中已保存的紀錄。
- 如果 Supabase 當天不穩，改用「複製完整 JSON」或「下載 JSON」作為備用畫面。

旁白：

> 最後，這筆 RHIR 可以被保存下來。這代表 Rent Unfiltered 不只是一次性的分析工具，也能累積可追溯、可比較的租屋風險資料，讓未來的租屋判斷更有依據。

字幕標籤：

- RHIR 保存至資料庫
- 可追溯的租屋風險資料

## 2:40-2:55 結尾

畫面：

- 回到 RRI 或 AI 報告重點摘要。
- 最後停在 Rent Unfiltered 首頁或專案名稱。

旁白：

> Rent Unfiltered 想做的不是替使用者決定要不要租，而是讓使用者在簽約前更早看見問題。當租屋資訊能被整理、比較與追溯，租屋就不必只靠經驗和運氣。

字幕標籤：

- 看懂風險，再做選擇
- Rent Unfiltered 透明租屋

## 不建議放太多的內容

- 名詞解釋頁：適合簡報或文件，不適合三分鐘主影片。
- 專案導引頁：容易像介紹網站，而不是展示系統價值。
- Chrome 擴充插件完整安裝流程：可以快速帶過，不建議花時間教安裝。
- Supabase 欄位細節：只要證明資料有保存，不要細講資料表。

## 拍攝前確認清單

- 新增表單一開始是空白。
- 一鍵補齊後會顯示 100%、31/31。
- Logo 可以回首頁，方便錄影時快速回到主畫面。
- AI 報告可以正常生成。
- Supabase 可以正常上傳或至少準備 JSON 備用畫面。
- 關掉通知、私人分頁、瀏覽器書籤列中的敏感資訊。

## 最終建議

就用這一套拍，不需要再加新的展示主題。

影片真正要讓評審記住的是：

> Rent Unfiltered 能把不完整的租屋資訊變成可追蹤、可比較、可解釋的風險判讀。
